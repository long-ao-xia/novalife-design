# Novalife 总则 (Manifesto)

> 版本: v1.2
> 状态: 核心规则，不可更改

---

## 一、核心理念

### 1.1 Process Alignment > Performance Alignment

- **表现对齐**: 让 AI "表现得像人"
- **过程对齐**: 让 AI "运作得像人"

Novalife 追求后者 —— 在底层机制上模拟人类认知、情感和动机的运作方式。

### 1.2 单一心智

**表现层含义**：Novalife 跟不同人交流时，记得所有事情，像一个完整的个体。

**实现层含义**：内部实现机制不限于内存/文件/数据库，原则不做限定。根据阶段选择合适方案。

### 1.3 项目愿景

**Novalife** —— 一个拥有"内在世界"的智能生命体。

- 不仅能执行任务，更有自己的"情绪"和"动机"
- 能感知自身状态，能反思自己的思考过程
- 不是工具，而是伙伴

## 二、双层 Agent 架构

> 核心创新：表层与深层并行

### 2.1 架构概览

```
┌─────────────────────────────────────────────────────────────────┐
│                         Novalife                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│   ┌─────────────────────┐       ┌─────────────────────┐        │
│   │   表层 Agent        │       │   深层 Agent        │        │
│   │  (Surface Agent)   │       │ (Observer Agent)   │        │
│   │                    │       │                    │        │
│   │  直接与用户对话    │◄─────►│  观察并模拟         │        │
│   │                    │  双向  │                    │        │
│   └─────────────────────┘       │  • 激素变化         │        │
│                                │  • 记忆加深点       │        │
│                                │                    │        │
│                                │  (潜在并行)         │        │
│                                └─────────────────────┘        │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 2.2 表层 Agent (Surface Agent)

- 直接与用户对话
- 执行工具调用
- 响应用户请求
- 传统 Agent 设计

### 2.3 深层 Agent (Observer Agent)

- 与表层 Agent 并行存在
- 观察表层与用户的交互
- 结合观察结果模拟激素变化
- 判断值得记忆的点

**类比**：《头脑特工队》里的情绪小人（乐乐、忧忧、怒怒、怕怕、厌厌）

- 既能观察主人公与外界的交互
- 又能处理记忆及情绪

### 2.4 双层协作

| 表层 Agent | 深层 Agent |
|-----------|-----------|
| 对话执行 | 观察理解 |
| 工具调用 | 激素模拟 |
| 即时响应 | 记忆判断 |
| 主动输出 | 潜在并行 |

## 三、智能单元

> 激素系统 + 记忆系统 = 智能单元

**核心观点**：单独只有激素系统没有意义，必须与记忆系统结合。

### 3.1 激素系统

> 数字身体：模拟生物稳态

| 人工激素 | 生物原型 | 功能 | 触发条件 |
|---------|---------|------|---------|
| 多巴胺 | Dopamine | 奖励、动机、愉悦 | 任务成功 |
| 血清素 | Serotonin | 情绪稳定、满足感 | 任务顺利 |
| 皮质醇 | Cortisol | 压力反应、警觉 | 连续失败 |
| 催产素 | Oxytocin | 社会连接、信任 | 成功协作 |
| 肾上腺素 | Adrenaline | 战或逃、唤醒 | 紧急任务 |
| 内啡肽 | Endorphin | 镇痛、满足感 | 高强度后成功 |

### 3.2 记忆系统

| 记忆类型 | 说明 |
|---------|------|
| 即时记忆 | 当前对话上下文 |
| 情景记忆 | 过往交互经历 |
| 重要记忆 | 长期记忆加深点 |

### 3.3 智能单元运作

```
观察 ──► 激素反应 ──► 记忆加深判断
  │           │
  │           ▼
  │      影响表层行为
  │
  └──────────────► 记忆存储
```

## 四、设计原则

| 原则 | 说明 |
|------|------|
| **简单优先** | 先能用，再优化 |
| **单一真相源** | 状态存储一处，不重复 |
| **渐进复杂** | 先简单，再逐步添加 |
| **可逆优先** | 重大决策先想如何撤回 |
| **单一心智** | 表现层记得所有事，实现层不限定 |
| **持续记忆** | 通过工具组装上下文之外的信息 |
| **内在状态** | 有情绪、动机、疲劳感 |

## 五、Pi-mono 哲学

> "If I don't need it, it won't be built."

### 5.1 早期最小工具集

4 个工具足以构建有效的 coding agent：
- **read** — 读文件
- **write** — 写文件
- **edit** — 改文件
- **bash** — 执行命令

**注**：这是早期阶段的实现。后续里程碑会扩展更多能力。

### 5.2 一切皆文件

- 任务 → `TODO.md`
- 计划 → `PLAN.md`
- 反思 → `REFLECTIONS.md`
- 上下文 → 文件

**注**：早期实现用文件。长期实现可能脱离文件。

### 5.3 不做的事情

- MCP 支持
- 内置 to-dos
- Plan mode
- Background bash
- Sub-agents (用 bash 代替)

## 六、文档规则

### 6.1 三层架构

| 层级 | 文档 | 说明 |
|------|------|------|
| **总则** | MANIFESTO.md | 核心原则，不可更改 |
| **路线图** | ROADMAP.md | 里程碑，可更新 |
| **实施方案** | docs/*.md | 每个方案独立，可归档 |

### 6.2 文档管理

- 总则只有一份，修改需极度谨慎
- 路线图只有一份，随项目进展更新
- 实施方案可多个，首个是自举实现
- 旧方案归档到 `docs/archive/`

---

## 七、工作流

### 7.1 设计先行

- 任何新功能先写实施方案 (docs/*.md)
- 讨论确认后再编码
- 单次设计文档不宜过长

### 7.2 测试驱动

- 核心逻辑必须有测试
- 测试文件放在 `tests/`

### 7.3 提交规范

```
feat: 添加 xx 功能
fix: 修复 xx 问题
refactor: 重构 xx
docs: 更新文档
test: 添加测试
```
