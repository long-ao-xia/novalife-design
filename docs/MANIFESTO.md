# Novalife 总则 (Manifesto)

> 版本: v1.0
> 状态: 核心规则，不可更改

---

## 一、核心理念

### 1.1 Process Alignment > Performance Alignment

- **表现对齐**: 让 AI "表现得像人"
- **过程对齐**: 让 AI "运作得像人"

Novalife 追求后者 —— 在底层机制上模拟人类认知、情感和动机的运作方式。

### 1.2 单一心智

一个大脑处理所有输入，无 Session 隔离。像人一样记住上下文。

---

## 二、设计原则

| 原则 | 说明 |
|------|------|
| **简单优先** | 先能用，再优化 |
| **单一真相源** | 状态存储一处，不重复 |
| **渐进复杂** | 先简单，再逐步添加 |
| **可逆优先** | 重大决策先想如何撤回 |

---

## 三、Pi-mono 哲学

> "If I don't need it, it won't be built."

### 3.1 最小工具集

4 个工具足以构建有效的 coding agent：
- **read** — 读文件
- **write** — 写文件
- **edit** — 改文件
- **bash** — 执行命令

### 3.2 一切皆文件

- 任务 → `TODO.md`
- 计划 → `PLAN.md`
- 反思 → `REFLECTIONS.md`
- 上下文 → 文件

Agent 不需要内置状态。

### 3.3 不做的事情

- MCP 支持
- 内置 to-dos
- Plan mode
- Background bash
- Sub-agents (用 bash 代替)

---

## 四、文档规则

### 4.1 三层架构

| 层级 | 文档 | 说明 |
|------|------|------|
| **总则** | MANIFESTO.md | 核心原则，不可更改 |
| **路线图** | ROADMAP.md | 里程碑，可更新 |
| **实施方案** | docs/*.md | 每个方案独立，可归档 |

### 4.2 文档管理

- 总则只有一份，修改需极度谨慎
- 路线图只有一份，随项目进展更新
- 实施方案可多个，首个是自举实现
- 旧方案归档到 `docs/archive/`

---

## 五、版本规则

- 文档版本用语义化版本 (v1.0)
- 每次重要更新记录变更日志
- 核心原则不向后兼容变更需 major version
