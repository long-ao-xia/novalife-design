## 一、核心理念

### 1.1 一句话概括

**像人脑一样运作的 AI Agent —— 一个脑子同时处理多件事，无 Session 隔离。**

### 1.2 Process Alignment > Performance Alignment

> 来自研究报告的核心洞见

- **表现对齐 (Performance Alignment)**: 让 AI "表现得像人"
- **过程对齐 (Process Alignment)**: 让 AI "运作得像人"

Novalife 追求后者 —— 在底层机制上模拟人类的认知架构、情感系统和动机机制。

### 1.3 单一心智

> "想象一个人，他在跟人聊天，同时回复手机上的消息，他一直用的是同一个脑子，而没有做任何隔离。"

这是 Novalife 最重要的设计决定：

| 方案 | 代表项目 | 特点 |
|------|---------|------|
| 强隔离 | OpenClaw, NanoClaw | 每个对话独立 Session/容器，更安全但复杂 |
| 无隔离 | Bub, Novalife | 单一上下文，简单，像人 |

## 二、设计原则

| 原则 | 说明 |
|------|------|
| **单一心智** | 一个大脑处理所有输入，无隔离 |
| **持续记忆** | 所有对话共享长期记忆 |
| **内在状态** | 有情绪、动机、疲劳感 (激素系统) |
| **渐进复杂** | 先简单，再逐步添加能力 |
| **单一真相源** | 状态存储在一处，不重复 |

## 七、关键设计决策

### 7.1 为什么不用 Session 隔离?

1. **像人一样**: 人的大脑不隔离处理的不同任务
2. **简单**: 不需要管理多个状态
3. **上下文连续**: 记得之前聊过什么

### 7.2 安全如何保证?

| 方案 | 说明 |
|------|------|
| 工具限制 | 不给危险工具 (rm -rf 等) |
| 操作审计 | 记录所有操作 |
| 可选容器 | 需要强隔离时可以加 |

### 7.3 上下文膨胀怎么办?

1. **Tape 压缩**: 定期压缩旧消息
2. **选择性记忆**: 只写重要的到 Long-term Memory
3. **注意力**: LLM 只关注当前相关的历史
