# Novalife 核心设计思想

> **一句话概括**: 像人脑一样运作的 AI Agent —— 一个脑子同时处理多件事，无 session 隔离。

## 核心理念

### 1. 无 Session 隔离

> "想象一个人，他在跟人聊天，同时回复手机上的消息，他一直用的是同一个脑子，而没有做任何隔离。"

这是 Novalife 最重要的设计决定：

- **传统设计** (OpenClaw/NanoClaw): 每个对话/群组有独立的 session、文件系统、记忆
- **Novalife 设计**: 单一心智，统一上下文，像人一样"一心多用"

### 2. Process Alignment > Performance Alignment

(来自研究报告)

- 不是让 AI "表现得像人"
- 而是让 AI "运作得像人" —— 底层机制模拟人类认知、情感、动机

### 3. 简单优先

- MVP 思维：小步快跑，能跑就行
- 单一大脑 > 多个隔离 Agent
- 单一状态 > 多个隔离上下文

## 设计原则

| 原则 | 说明 |
|------|------|
| **单一心智** | 一个大脑处理所有输入，无隔离 |
| **持续记忆** | 所有对话共享长期记忆 |
| **内在状态** | 有情绪、动机、疲劳感 |
| **渐进复杂** | 先简单，再逐步添加能力 |
| **单一真相源** | 状态存储在一处，不重复 |

## 与其他系统的对比

| 特性 | OpenClaw | NanoClaw | Bub | Novalife |
|------|----------|----------|-----|----------|
| Session 隔离 | ✅ 多session | ✅ per-group | ✅ | ❌ 无 |
| 架构复杂度 | 高 | 低 | 中 | 低 |
| 容器隔离 | 可选 | ✅ 强制 | 可选 | 可选 |
| 状态模型 | 多session | 多容器 | tape | 单一心智 |
| 技能系统 | ✅ | ✅ | ✅ | ✅ |

## 架构方向 (待定)

```
┌─────────────────────────────────────┐
│         单一心智 (Novalife)          │
├─────────────────────────────────────┤
│  输入层 ──▶ 理解 + 路由 ──▶ 输出      │
│     │              │                │
│     ▼              ▼                │
│  记忆层 ◀─────── 激素状态            │
│     │              │                │
│     ▼              ▼                │
│  工具层 ◀─────── 动机系统            │
└─────────────────────────────────────┘
```

### 核心模块 (MVP)

1. **输入路由器** - 接收所有消息，统一处理
2. **单一记忆系统** - 所有对话共享
3. **激素状态机** - 情绪/动机动态变化
4. **工具执行层** - 能力扩展

### 后续可扩展

- 多模态输入
- 长期记忆巩固
- 社会认知模块

## 命名空间

- 无需为不同对话创建隔离空间
- 所有上下文在同一棵记忆树中
- 通过"注意力"机制区分当前焦点
